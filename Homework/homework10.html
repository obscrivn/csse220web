How to Import a ZIP File into Eclipse

Follow these steps to import a ZIP file containing an Eclipse project:

  1. Download the ZIP File (containing a project) to your computer but do not place in the eclipse workspace folder and do not unzip it: Click here to donwload ZIP
  2. Open Eclipse: Do not create a new project because the ZIP file contains an existing project.
  3. Import the Project:
    1. Click on File in the menu bar and select Import....
    2. In the Import Wizard, expand the General category and select Existing Projects into Workspace. Click Next.
    3. Select the Select archive file option and click the Browse... button
    4. Navigate to the location where you saved the ZIP file, select it, and click Open
    5. The wizard will display the project(s) contained in the ZIP file. Make sure the desired project is checked
    6. Check if the box Copy projects into workspace is selected
    7. Click Finish to complete the import process
  4. Verify the Imported Project: The project should now appear in your Package Explorer. Double-click it to open and review its contents.

HWK 10 Part 1 – Ball/Strike Counter

Overview

  • You have a simple GUI framework that tracks balls and strikes. You need to add buttons, ActionListeners, and variables to make it function. You may add classes or changes as needed. The updateLabel method is provided to update the display.

Part 1 To Do

  1. Make the Ball and Add Strike buttons appear in the app’s window as shown below
  2. The buttons must work as follows:
    • The Add Ball button, when clicked, increments the Balls count label. If Balls reaches 4, reset both Balls and Strikes to 0.
    • The Add Strike button, when clicked, increments the Strikes count label. If Strikes reaches 3, reset both Balls and Strikes to 0.
    • Update the labels on every click to reflect the new counts.

Overview

  1. Implement the app in the linearLightsOut package.
  2. The app presents an array of buttons randomly initialized to X or O (50% chance each).
  3. Clicking a button toggles its symbol and that of its left/right neighbors (if they exist).
  4. Edge buttons toggle only themselves and their single neighbor (no wrapping).
  5. The goal is to make all buttons show the same symbol (all Xs or all Os).
  6. When the user wins, notify them and end the game.

This starter package gives minimal scaffolding; use prior homework or the JButton API as needed. Ask for help if stuck.
