How to Import a ZIP File into Eclipse

Follow these steps to import a ZIP file containing an Eclipse project:

  1. Download the ZIP File (containing a project) to your computer but do not place in the eclipse workspace folder: Click here to donwload ZIP
  2. Open Eclipse: You do not need to create a new project because the ZIP file contains an existing project.
  3. Import the Project:
    1. Click on File in the menu bar and select Import....
    2. In the Import Wizard, expand the General category and select Existing Projects into Workspace. Click Next.
    3. Select the Select archive file option and click the Browse... button
    4. Navigate to the location where you saved the ZIP file, select it, and click Open
    5. The wizard will display the project(s) contained in the ZIP file. Make sure the desired project is checked
    6. Check if the box Copy projects into workspace is selected
    7. Click Finish to complete the import process
  4. Verify the Imported Project: The project should now appear in your Package Explorer. Double-click it to open and review its contents.

Problem #2 ‚ûï

addFraction(int num1, int den1, int num2, int den2)

Adds two fractions and gives you the decimal result as a double.

üí° Hint (click to expand)

The key idea is to compute each fraction as a double and then add them.

  • Convert the numerator to a double before dividing!
    (double) num1 / den1

Note: Be careful about integer division!

If you write: 

(double)(num1 / den1)

Java will do integer division first, since both num1 and den1 are integers.

For example:

(double)(1 / 2)  // 1 / 2 is 0 as an int ‚Üí becomes 0.0

That loses the decimal part.

Instead, cast before dividing: 

(double) 1 / 2  // 1 becomes 1.0, so 1.0 / 2 = 0.5

This forces Java to use decimal (floating-point) division, which is what you want.

Problem #3 üîç

largestSingleDigit(long input)

Scans a whole number and reports the biggest digit hiding inside it.

üí° Hint 1: Module + Division

To extract the last digit of a number, use the modulo operator (%) with 10. For example, if the input is 15342, then 15342 % 10 gives 2 (the last digit). After processing the last digit, use integer division (/) by 10 to remove it. Repeat these steps until the number is reduced to 0. Compare each digit to determine the largest one. Here is the workflow:

  • Use the modulo operator % 10 to get the last digit.
  • Compare it to your current "largest digit so far".
  • Use integer division / 10 to remove the last digit.
  • Repeat until the number becomes 0.
üí° Hint 2: Casting Pitfalls

If you write something like: 

int digit = (int) input % 10;

Java will cast input to an int before the % 10 happens. For very large long values, this cast can overflow and produce strange numbers.

Instead, make sure you take the last digit first, then cast that small value: 

int digit = (int) (input % 10);

This way:

  • input % 10 gives a value between 0 and 9 (always safe)
  • Then you cast that tiny digit to int
This prevents overflow and makes your method work correctly even for very large numbers.
