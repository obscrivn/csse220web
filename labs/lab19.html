4. Ending the Game

private void onEndGame() {
            String player = JOptionPane.showInputDialog(this, "Enter your name", "Player");
            if (player == null || player.isBlank()) player = "Anonymous";
          
            ScoreManager.saveScore(player, score);
            showLeaderboard();
          
            score = 0;
            scoreLabel.setText("Score: 0");
          }

5. Displaying the Leaderboard

private void showLeaderboard() {
            List lines = ScoreManager.loadScores();
              StringBuilder html = new StringBuilder(
                "<html><h2>Leaderboard</h2><ol>"
            );
            for (int i = 0; i < lines.size() && i < 5; i++) {
              String line = lines.get(i);
                String[] parts = line.split(",", 2);
                html.append("<li>")
                    .append(parts[0])    // player name
                    .append(": ")
                    .append(parts[1])    // score
                    .append("</li>");
            }
            html.append("</ol></html>");
          
            JOptionPane.showMessageDialog(
              this,
              new JLabel(html.toString()),
              "Top Scores",
              JOptionPane.PLAIN_MESSAGE
            );
          }
  • ScoreManager.loadScores(): returns raw “name,score” lines from the file
  • StringBuilder + HTML: builds a simple ordered list for Swing’s JLabel
  • showMessageDialog: shows the leaderboard in a modal dialog
  • Limits to top 5 entries

ScoreManager

The ScoreManager class handles all file I/O related to persisting and retrieving game scores It exposes two static methods:

  1. saveScore(String player, int score)
  2. loadScores()

1. File Path Constant


          private static final String SCORES_FILE = "scores.txt";
  • Why a constant? Centralizes the filename so if you ever change where scores are saved, you update in one place
  • Relative path: The file lives in the working directory (usually your project root or wherever the JVM was launched)

2. Saving a Score


          public static void saveScore(String player, int score) {
              try (PrintWriter out = 
                       new PrintWriter(
                         new FileWriter(SCORES_FILE, true)  // append mode
                       )
                 ) {
                  out.println(player + "," + score);
              } catch (IOException e) {
                  System.err.println("Error saving score: " + e.getMessage());
              }
          }
  • Append mode: new FileWriter(SCORES_FILE, true) opens the file without erasing existing lines
  • PrintWriter: wraps the FileWriter to provide println(...) convenience
  • Try block ensures out.close() is called automatically, flushing and releasing the file handle
  • Exception handling: any IOException (disk full, permission denied) is caught and logged to System.err without crashing the game

3. Loading Scores


          public static List<String> loadScores() {
              List<String> lines = new ArrayList<>();
              File file = new File(SCORES_FILE);
              if (!file.exists()) {
                  return lines;  // no scores file yet,  return empty list
              }
          
              try (BufferedReader in = 
                       new BufferedReader(new FileReader(file))
                 ) {
                  String line;
                  while ((line = in.readLine()) != null) {
                      lines.add(line.trim());
                  }
              } catch (IOException e) {
                  System.err.println("Error loading scores: " + e.getMessage());
              }
          
              return lines;
          }
  • Return type: List<String> where each element is a line in "player,score" format
  • Existence check: if (!file.exists()) avoids a FileNotFoundException on first run
  • BufferedReader: efficient line-by-line reading, auto-closed by try block
  • Trim: line.trim() removes stray whitespace before/after each line
  • Error handling: logs I/O errors but returns whatever lines were successfully read (or an empty list)

5. Best Practices

  • Single Responsibility: File I/O is isolated in ScoreManager; your UI code does not need to know the details
  • Robustness: All exceptions are caught and reported; the game continues even if the disk is full or the file is missing
  • Maintainability: Using constants and standard Java I/O patterns makes the code easy to understand and update
