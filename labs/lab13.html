Event Handling Patterns

In this lab, you will practice four different ways to handle events in Java Swing. Each version works, but the code organization is a little different.

  • MyCounter: use an ActionListener that calls a helper method.
    The listener responds to the button click, then calls a separate method such as increment() to handle the action.
  • MyAccumulator: use an ActionListener with no helper method.
    Write the logic directly inside the listener.
  • Refactoring MyCounter: use an ActionListener with a separate outer listener class.
    The event-handling code will live in its own class.
  • ThreeButtonCounter: use an ActionListener with an inner listener class.
    One inner class handles multiple buttons.

Each approach works, but they differ in readability, flexibility, and reuse.

Be ready to compare: Which one is easiest to understand? Which one would scale better?

01. ActionListener My Counter Submission

Adding ActionListener to the Button

Let’s make the button do something! Whenever the “Count” button is clicked, it should call the increment() method that updates the number in the text field.

  1. Inside the constructor (after adding your components), register the listener for the button using a lambda expression: 
    button.addActionListener(e -> increment());

    The expression e -> increment() means: “When the button is clicked, run the increment() method.”

  2. Implement the increment() method in your class. It should:
    1. Increase the counter value by 1
    2. Update the text field to display the new count
    public void increment() {
    // update count
    // update text field

}
💡 Hint: Why Not Declare count Inside increment()?

If you declare int count = 0; inside the increment() method, it will reset to zero each time you click the button. Using an instance variable ensures the value is remembered between clicks.

💡 Hint: One-Line vs Multi-Line Lambdas

A one-line lambda (like e -> increment()) does not need braces or semicolons. If you want to perform multiple actions, use braces and semicolons inside: 

button.addActionListener(e -> {
    count++;
    field.setText(Integer.toString(count));
});

💡 Hint: OOP Design Principle

This approach follows encapsulation: the button only triggers an event, while the MyCounter decides what to do (update its count and text field). Separating logic like this is good design and prepares you for the Model–View–Controller pattern.
