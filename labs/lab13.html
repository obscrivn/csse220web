Part 1. Adding ActionListener to the Button - Lambda

Now that you have your layout ready, let‚Äôs make the button do something! Whenever the ‚ÄúCount‚Äù button is clicked, it should call the increment() method that updates the number in the text field.

  1. Inside the constructor (after adding your components), register the listener for the button using a lambda expression: 
    button.addActionListener(e -> increment());

    The expression e -> increment() means: ‚ÄúWhen the button is clicked, run the increment() method.‚Äù

  2. Implement the increment() method in your class. It should:
    1. Increase the counter value by 1
    2. Update the text field to display the new count
    private void increment() {
    count++;
    text.setText(Integer.toString(count));
}

    üí° Make sure your count and text are declared as instance variables (outside the constructor), so they can be accessed inside increment(). 

    private int count = 0;
private JTextField text; // declared at class level

üí° Hint: Why Not Declare count Inside increment()?

If you declare int count = 0; inside the increment() method, it will reset to zero each time you click the button. Using an instance variable ensures the value is remembered between clicks.

üí° Hint: One-Line vs Multi-Line Lambdas

A one-line lambda (like e -> increment()) does not need braces or semicolons. If you want to perform multiple actions, use braces and semicolons inside: 

button.addActionListener(e -> {
    count++;
    text.setText(Integer.toString(count));
});

Part 2: Using a Custom ActionListener Class

In this part, you will replace the lambda expression with a custom ActionListener class. This will help you see how events can be handled by separate objects in larger programs.

  1. Comment out your lambda listener line: 
    // button.addActionListener(e -> increment());
  2. Create a new class named MyCounterActionListener that implements ActionListener from the java.awt.event package.
    This class should:
    • Have a private field of type MyCounter
    • Store a reference to the counter in its constructor
    • Call increment() on the counter when the button is clicked
    import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;

public class MyCounterActionListener implements ActionListener {

    private MyCounter counter;

    public MyCounterActionListener(MyCounter counter) {
        this.counter = counter;
    }

    @Override
    public void actionPerformed(ActionEvent e) {
        counter.increment();
    }
}
  3. In your MyCounter constructor, create an instance of this listener and register it with the button: 
    MyCounterActionListener listener = new MyCounterActionListener(this);
button.addActionListener(listener);

When the button is clicked, Java will automatically create an ActionEvent and call your actionPerformed() method inside MyCounterActionListener.

üí° Hint: Why Pass this?

The this keyword refers to the MyCounter object itself. The listener needs this reference so it can call increment() on the panel when the event occurs. You are passing the whole counter object, not just the button.

üí° Hint: OOP Design Principle

This approach follows encapsulation ‚Äî the button only triggers an event, while the MyCounter decides what to do (update its count and text field). Separating logic like this is good design and prepares you for the Model‚ÄìView‚ÄìController pattern.
