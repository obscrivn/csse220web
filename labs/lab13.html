02. ActionListener Practice

Part 1. Adding ActionListener to the Button - Lambda

Now that you have your layout ready, let‚Äôs make the button do something! Whenever the ‚ÄúCount‚Äù button is clicked, it should call the increment() method that updates the number in the text field.

  1. Inside the constructor (after adding your components), register the listener for the button using a lambda expression: 
    button.addActionListener(e -> increment());

    The expression e -> increment() means: ‚ÄúWhen the button is clicked, run the increment() method.‚Äù

  2. Implement the increment() method in your class. It should:
    1. Increase the counter value by 1
    2. Update the text field to display the new count
    private void increment() {
    count++;
    text.setText(Integer.toString(count));
}

    üí° Make sure your count and text are declared as instance variables (outside the constructor), so they can be accessed inside increment(). 

    private int count = 0;
private JTextField text; // declared at class level

üí° Hint: Why Not Declare count Inside increment()?

If you declare int count = 0; inside the increment() method, it will reset to zero each time you click the button. Using an instance variable ensures the value is remembered between clicks.

üí° Hint: One-Line vs Multi-Line Lambdas

A one-line lambda (like e -> increment()) does not need braces or semicolons. If you want to perform multiple actions, use braces and semicolons inside: 

button.addActionListener(e -> {
    count++;
    text.setText(Integer.toString(count));
});

Part 2: Using a Custom ActionListener Class

In this part, you will replace the lambda expression with a custom ActionListener class. This will help you see how events can be handled by separate objects in larger programs.

  1. Comment out your lambda listener line: 
    // button.addActionListener(e -> increment());
  2. Create a new class named MyCounterActionListener that implements ActionListener from the java.awt.event package.
    This class should:
    • Have a private field of type MyCounter
    • Store a reference to the counter in its constructor
    • Call increment() on the counter when the button is clicked
    import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;

public class MyCounterActionListener implements ActionListener {

    private MyCounter counter;

    public MyCounterActionListener(MyCounter counter) {
        this.counter = counter;
    }

    @Override
    public void actionPerformed(ActionEvent e) {
        counter.increment();
    }
}
  3. In your MyCounter constructor, create an instance of this listener and register it with the button: 
    MyCounterActionListener listener = new MyCounterActionListener(this);
button.addActionListener(listener);

When the button is clicked, Java will automatically create an ActionEvent and call your actionPerformed() method inside MyCounterActionListener.

üí° Hint: Why Pass this?

The this keyword refers to the MyCounter object itself. The listener needs this reference so it can call increment() on the panel when the event occurs. You are passing the whole counter object, not just the button.

üí° Hint: OOP Design Principle

This approach follows encapsulation ‚Äî the button only triggers an event, while the MyCounter decides what to do (update its count and text field). Separating logic like this is good design and prepares you for the Model‚ÄìView‚ÄìController pattern.

03. Inheritance and Method Overriding: GameEntity and PlayerEntity

In this part of the lab, you will explore how inheritance and method overriding work in Java by building a simple 2D game entity system.

You will first create a general GameEntity class that represents any moving object, and then extend it with a PlayerEntity subclass that adds new data and behavior.

Step 1: Define Fields in GameEntity

Create a class GameEntity with the following protected fields (so they are visible to subclasses):

  • protected int x, y; ‚Äì the current position of the entity
  • protected double speed; ‚Äì how much the entity moves horizontally per frame
  • protected int health; ‚Äì the health or hit points
  • protected static final int SCREEN_WIDTH = 800; ‚Äì screen boundary constant

Step 2: Create the Constructor

Write a constructor that takes 4 parameters and assigns them to the fields: 

public GameEntity(int startX, int startY, double startSpeed, int startHealth) {
    this.x = startX;
    this.y = startY;
    this.speed = startSpeed;
    this.health = startHealth;
}

‚úÖ Recall: The this keyword distinguishes between the instance field (like this.x) and the parameter startX.

Step 3: Override toString()

The toString() method defines how the object is displayed when printed. Override it to show all the important data: 

@Override
public String toString() {
    return String.format("Entity@(%d,%d) speed=%.2f health=%d", x, y, speed, health);
}

Example: Testing toString() Override

The toString() method is automatically called when you print an object. After overriding it in GameEntity, printing the object shows useful details instead of a memory address. 

GameEntity e = new GameEntity(100, 50, 10.0, 100);
System.out.println(e);

// Output:
Entity@(100,50) speed=10.00 health=100

Tip: Use String.format to control number formatting (%.2f = 2 decimal places).

Step 4: Add the update() Method

The update() method simulates a single "frame" of movement. Each call will move the entity horizontally and ensure it stays on screen. 

public void update() {
    // Move horizontally by speed
    x += (int) speed;

    // Clamp x between 0 and SCREEN_WIDTH
    x = Math.max(0, Math.min(SCREEN_WIDTH, x));
}

üß© Clamping means restricting a value within a certain range. If x goes below 0 ‚Üí set it to 0. If x goes beyond 800 ‚Üí set it to 800. Otherwise, keep the same value.

Step 5: Test GameEntity in main()

public static void main(String[] args) {
    GameEntity e = new GameEntity(790, 100, 15.0, 100);
    System.out.println(e);
    e.update();       // moves to x = 805 but gets clamped to 800
    System.out.println(e);
}

‚úÖ Expected Output:

Step 6: Create a Subclass PlayerEntity

Now let‚Äôs create a subclass that extends GameEntity to represent the player. 

public class PlayerEntity extends GameEntity {

Add these extra fields specific to the player:

  • private String playerName; ‚Äì the player‚Äôs name
  • private int score; ‚Äì the player‚Äôs current score
  • Flags to simulate key input:
    public boolean upPressed = true, downPressed = false;

The constructor should take 5 parameters and call the GameEntity constructor with super(...): 

public PlayerEntity(String name, int startX, int startY, double speed, int health) {
    super(startX, startY, speed, health);  // calls GameEntity constructor
    this.playerName = name;
    this.score = 0;
}

‚úÖ Note: The super(...) call must be the first line of the constructor. It sets up the inherited fields from the GameEntity class.

Step 7: Override update()

The player moves horizontally (inherited behavior) but also vertically based on key flags. This is a perfect time to demonstrate method overriding: 

@Override
public void update() {
    // Step 1: reuse the base movement
    super.update();

    // Step 2: add vertical movement
    if (upPressed) {
        y -= 5;
    }
    if (downPressed) {
        y += 5;
    }

    // Step 3: increase score if passed a threshold
    if (x > 400) {
        score += 10;
    }
}

Notice how super.update() lets us reuse logic from GameEntity instead of rewriting the same code. This is one of the biggest benefits of inheritance.

Step 8: Override toString()

Make the player‚Äôs version print player-specific information. 

@Override
public String toString() {
    return String.format("%s@(%d,%d) score=%d", playerName, x, y, score);
}

Step 9: Test PlayerEntity in main()

public static void main(String[] args) {
    GameEntity e = new PlayerEntity("Alice", 390, 50, 20.0, 100);
    System.out.println(e);
    e.update();  // runs PlayerEntity.update() because of overriding
    System.out.println(e);
}

‚úÖ Expected Output:
Alice@(390,50) score=0
Alice@(410,45) score=10

üéØ This happens because:

  • x increased from 390 ‚Üí 410 (20 units)
  • y decreased by 5 because upPressed = true
  • score increased by 10 since x > 400

üí° Hint: Overriding vs. Overloading

Overriding = same method name and parameters, new behavior in a subclass.
Overloading = same method name, but different parameter list (in the same class).
