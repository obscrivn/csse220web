üí° Hint 7: Why use / 3.0 instead of / 3?

If your parameters are already double, then / 3 and / 3.0 give the same result: Java will automatically treat the 3 as 3.0 during the calculation.

But if your parameters are int, then / 3 triggers integer division, which chops off decimals. Example:

(2 + 3 + 8) = 13
13 / 3     = 4     ‚Üê integer division üò¨

Using / 3.0 forces Java to do floating-point division, which keeps the decimal part:

(2 + 3 + 8) = 13
13 / 3.0   = 4.3333...  ‚Üê floating-point division üòå

TL;DR: / 3.0 is the safe option that always gives you the correct decimal average.

üí° Hint: bunchOfOnes() Start with result = 0, then loop length times. In each loop, first do result = result * 10; to shift left, then result = result + 1; to add a new 1 digit at the end.
üí° Hint 1: Growing the number

If you have 11 and want 111, think:
11 * 10 = 110, then add 1 ‚Üí 111.

Each time through the loop, the current value is multiplied by 10 * to shift its digits left, and then 1 is added to append a new 1 at the end. * For instance, if current is 11, multiplying by 10 produces 110, and adding 1 * results in 111.
