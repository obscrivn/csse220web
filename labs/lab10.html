Exercise 2: Graphics

In this lab you will learn about Java interfaces and how they can help reduce duplicate code by defining a common contract. You will create an interface Pet with three common public methods and modify the provided classes (Cat, Dog, and Fish) so that they implement this interface. Then, you will refactor the rest of the code (including Zookeeper and PetMain).

Download the zip file: pets.zip Import it as Archive file under your src folder - it will be shown as package named pets. alternatively, you can unzip and import files separately to your own package.

Note - if you notice an external file MACOX shown also, please delete it.

Run PetMain to see the current output. Use your UML plugin to see the classes schema and their relationships.

Task 1: Create the Pet Interface

Create an interface called Pet that declares the following three methods:

  • public void eatFood();
  • public boolean isEating();
  • public String getName();

Note: Even though your original method names in the classes are eatFood(), isEating(), and getName(), ensure that the interface's method signatures exactly match these methods.

Task 2: Modify the Pet Classes

Update the classes Cat, Dog, and Fish to implement the Pet interface. For each class:

  • Add implements Pet to the class declaration
  • Ensure that the methods eatFood(), isEating(), and getName() have the same signatures as declared in the interface

Task 3: Refactor Zookeeper

To remove duplicate code, modify the Zookeeper class so that it no longer has separate feedCat, feedDog, and feedFish methods. Instead, create a single method that accepts a Pet object. Recall - (Dog pet) vs (Pet pet).

Task 4: Update PetMain

Refactor the PetMain class to use a single ArrayList of type Pet instead of separate lists for each pet type. This will allow you to iterate through all pets in a unified way.

Note: In this PetMainclass, you'll notice that pet collections are declared as List<Pet> rather than a specific implementation. This practice, known as "programming to an interface," promotes flexibility and maintainability. For example, by initializing the list as new ArrayList<>(), you can later switch to a different list implementation (like LinkedList) without having to change the rest of your code. Use the same format whe you refactor: private List<Pet> pets

Note: The original PetMain had separate lists for cats, dogs, and fishes. Refactoring to a single list of type Pet reduces duplicate code and makes it easier to work with all pets uniformly.
