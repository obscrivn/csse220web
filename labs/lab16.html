Lab 1: Exception Handling Basics

In this lab you will see how Java throws and catches two common runtime errors: ArithmeticException (division by zero) and NullPointerException. You should wrap it in a try/catch block so the program can recover and continue.

1. Create a new class ExceptionDemo

  1. Define the class and main method: 
    public class ExceptionDemo {
      public static void main(String[] args) {
          // TODO: add code here
      }
  }
  2. Inside main, add a try/catch for division by zero: 
    try {
      int a = 10;
      int b = 0;
      System.out.println("Result: " + (a / b));
  } catch (ArithmeticException e) {
      System.err.println("Caught ArithmeticException: " + e.getMessage());
  }
  3. Below it, add a second try/catch for a null reference: 
    try {
      String s = null;
      System.out.println("Length: " + s.length());
  } catch (NullPointerException e) {
      System.err.println("Caught NullPointerException: " + e.getMessage());
  }
  4. Finally, print a message after both blocks: 
    System.out.println("Program continues after handling exceptions.");

2. Run and Observe

  • Compile and run ExceptionDemo
  • Confirm you see messages for each caught exception instead of a crash
  • Notice that the final print runs, showing the program recovered

3. Reflection

  • What happens if you remove the catch block for ArithmeticException?
  • Why are these exceptions unchecked?

Lab 2: Image Loading with Exception Handling

Next, you will build a simple Swing application that loads an image file in a custom panel, catches any IOException if the file is missing or unreadable, and displays a fallback message.

1. Project Setup

  1. You can use the same package from graphics Lab 15. Create two folders under src: images (for PNG/JPG) and audio
  2. Download and place ths image (Link) into src/images/

2. Write ImagePanel

ImagePanel will extend JPanel

Add a field private BufferedImage image;

Create a constructor:


              public ImagePanel() {
                    image = ImageIO.read(new File("src/images/imagename.png"));        
                    setPreferredSize(new Dimension(400, 300));
            }

Add try and catch:


                  try {
                      image = ImageIO.read(new File("src/images/imagename.png"));
                  } catch (IOException e) {
                      System.err.println("Error loading image: " + e.getMessage());
                      image = null;
                  }

Why do you think we need image = null?

Add paintComponent

@Override protected void paintComponent(Graphics g) { super.paintComponent(g); if (image != null) { int x = (getWidth() - image.getWidth()) / 2; int y = (getHeight() - image.getHeight()) / 2; g.drawImage(image, x, y, this); } else { g.setColor(Color.RED); g.drawString("Image not found", 20, 20); } }
