02. Polymorphism

Follow instructions and answer questions at the end of the lab - add your answers as comments in java file

Lab: Exploring Polymorphism, Upcasting & Downcasting

In this lab you will work with a simple class hierarchy to see how Java chooses which print() method to call at runtime (polymorphism), how upcasts let you treat a subclass as its superclass, and how downcasts let you recover subclass‐specific behavior, sometimes safely and sometimes resulting in errors if misused. Note that @Override annotation is optional but strongly recommended whenever you intend to override a superclass or interface method.

1. Setup & Code

Copy the following into PolymorphismDemo.java


public class PolymorphismDemo {
    public static void main(String[] args) {
        A x = new D();
        x.print();               // (1)
        D x2 = (D)x;
        x2.print();              // (2)
        x2.printC();             // (3)
        B x3 = (B)x;
        // x3.printC();         // (4) 
    }
}

class A {
    public void print() { System.out.println("A"); }
}

class B extends A {
    @Override public void print() { System.out.println("B"); }
}

class C extends B {
    @Override public void print() { System.out.println("C"); }
    public void printC() { print(); }
}

class D extends C {
    @Override public void print() { System.out.println("D"); }
}

Note: removing print() in D will cause Java to invoke the next override up the chain, demonstrating dynamic dispatch
