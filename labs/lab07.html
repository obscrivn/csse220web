Exercise 2: Design A/B/Main Classes

Step 1: Analyze the UML Diagram

Below is a UML diagram (classABC.png) that outlines three classes: A, B, and Main. Your first task is to recreate this diagram using PlantUML. Observe the relationships, fields, and methods shown, and then replicate them in your PlantUML code.

Once you have replicated the diagram in PlantUML, proceed to the next steps to implement these classes in Java.

Step 2: Create Class B

From your UML diagram, you can see that class B has an integer field called count and methods to manipulate it. Follow the instructions below to build this class in Java.

  1. Field: Declare a private integer field named count.
  2. Constructor: Create a no-argument constructor that initializes count to 0.
  3. Setter Method: Add a method setValue(int value) to update the count field.
  4. Getter Method: Add a method getCount() that returns the current value of count.

Why is this setup important?

  • Encapsulation: The count field is private, ensuring it can only be modified through the class methods (setValue and getCount). This prevents external classes from making unintended changes.
  • Constructor: Initializing count to 0 provides a sensible default. You can change this if your design calls for a different initial value.
  • Setter and Getter: These methods follow the standard Java naming convention for manipulating private fields. This pattern is part of the broader concept of encapsulation and helps maintain the integrity of your data.

Step 2: Create Class A

Class A has the following characteristics:

  • A private String name field to store the name
  • A private B object (b) to represent its association with class B
  • A constructor that initializes the name and creates a new instance of B
  • A method setBValue(int value) that uses the B object to set a value
  • Getter methods: getName() and getBValue() to retrieve the name and the current value from the B object

Step 3: Create Class Main

The Main class ties together the classes A and B. The Main class is responsible for:

  • Maintaining a list of A objects
  • Initializing these objects in the constructor
  • Providing a method to update the B value for all A objects
  • Printing a report that shows each A object's name and the current value from its associated B object

Additional Instructions

  2. Constructor:

  3. Method setAllBValuesTo3():
    Iterate over the list and call setBValue(3) on each A object. This method updates each associated B's value to 3.
  4. Method printReport():
    Iterate over the list and print each A object's name along with the current value of its associated B (retrieved via getBValue()).
  5. Main Method:
    In the main method, create an instance of Main, update the B values by calling setAllBValuesTo3(), and then print the report.
