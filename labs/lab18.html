01. Abstract and Interface

In this section you‚Äôll combine an interface and an abstract class to see how they divide responsibilities between capability (interface) and structure (abstract class).

Step 1. Create an Interface

public interface Renderable {
    void render();
}

Step 2. Create an Abstract Class Entity that Implements the Interface

public abstract class Entity implements Renderable {
    private String id;

    public Entity(String id) { this.id = id; }

    public String getId() { return id; }

    @Override
    public void render() {
        System.out.println("Rendering " + id);
    }

    public abstract void update();
}
üí° Why abstract and interface together?

Renderable defines a capability (something that can be rendered). Entity provides shared state (id) and partial implementation (render()). Subclasses only supply the details by overriding update().

Step 3. Create a Subclass Player

public class Player extends Entity {
    public Player(String id) { super(id); }

    @Override
    public void update() {
        System.out.println("Updating " + getId());
    }
}

Step 4. Test in main()

public class Demo {
    public static void main(String[] args) {
        Entity p = new Player("Alice");
        p.render();
        p.update();
    }
}

Key Takeaways

  • Interfaces define what classes can do ‚Üí capabilities.
  • Abstract classes define how things partly work ‚Üí shared code + state.
  • Polymorphism lets one reference call different implementations at run-time.
  • Upcasting is automatic; downcasting requires an explicit cast and careful checks.
