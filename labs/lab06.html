01. Encapsulation and Refactoring Submission

🎮 Scenario

In your Minecraft world, players explore caves and dungeons to collect loot. When a player finds a Treasure Chest, they can store items inside it.

Each item can be something like:

  • a sword
  • a diamond

Every item has:

  • a name
  • a gold value, such as 100 or 10

The TreasureChest should be able to:

  • store items
  • add items
  • remove items
  • count how many items it has
  • calculate the total value of all items
  • enforce a maximum capacity

Each item should also be able to determine whether it is rare:

  • if its value is greater than 50, it is considered rare

Right now, the code works, but the design is not great.

This design is awkward because one item is split into separate pieces of data. Also, the chest is deciding whether an item is rare, even though rarity really belongs to the item itself.

Your job in this lab is to refactor the design by creating an Item class and updating TreasureChest to store ArrayList<Item>.

⚠️ Bad Design

The current design has problems:

  • One item is stored as separate pieces of data instead of as one object
  • TreasureChest is doing item-specific work
  • The design will become messier if items later need more data

Example of the current awkward design: 

chest.addItem("Diamond", 100);
chest.isRare("Diamond");

A better design would treat one item as one object: 

Item diamond = new Item("Diamond", 100);
chest.addItem(diamond);
diamond.isRare();

💡 Important Design Notes

Why move isRare() into Item?

Rarity is something about one item, not about the chest. The chest should manage items. The item should know item-specific information about itself.

How should isRare() work?

Do not add a separate rare field. Instead, compute rarity from the value. 

public boolean isRare() {
    return this.value > 50;
}

This is better because we do not want to store extra data if we can compute it from existing data.

🧠 Do not get stuck

1. Updating addItem

Old version:

public void addItem(String name, int value)

New version:

public void addItem(Item item)

You are no longer adding separate pieces of data. You are adding one object.

2. Updating getTotalValue()

Old idea:

for (int value : itemValues) {
    total += value;
}

New idea:

for (Item item : items) {
    total += item.getValue();
}

3. Updating removeItem

In the bad design, removal used a name. In the refactored design, remove the actual object: 

items.remove(item);

4. Updating GameApp

Old version:

chest.addItem("Diamond", 100);

New version:

Item diamond = new Item("Diamond", 100);
chest.addItem(diamond);
