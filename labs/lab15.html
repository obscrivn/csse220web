Step 1: Load a Sprite Image to the Ball Class

So far, the ball is drawn as a circle. Now we will upgrade it into a sprite. A sprite is an image used to represent a game object.

Add a field to store the sprite: 

private BufferedImage sprite;

A BufferedImage is an image stored in memory. Think of it as pixel data that Java can draw on the screen.

ImageIO is a Java utility class that reads image files such as PNG or JPG. It converts the image file into a BufferedImage. 

sprite = ImageIO.read(
      Ball.class.getResource("tennis.png"));

This can fail if the image is missing, the file name is wrong, the path is wrong, or the image file cannot be read. Java will force us to catch it. 

try {
  sprite = ImageIO.read(
    Ball.class.getResource("tennis.png"));
} catch (IOException | IllegalArgumentException e) {
  sprite = null;
}

In this example, if the image does not load, we set sprite to null. Then we can draw a simple shape instead of crashing the program. We also use Ball.class to use the Ball class as the reference point.

The single | in the catch statement means either exception type can be caught. This is called multi-catch. It is different from boolean OR ||.

Step 2: Draw Sprite Image to the Ball

Inside Ball's draw() method call drawImage() to load the sprite. If the image does not load, draw a circle as a backup. 

if (sprite != null) {

 }

Step 3: Load a Background Image

The background belongs to the drawing surface, not the player. Add it to GameComponent, because GameComponent controls what appears on the screen.

Add a field: 

private BufferedImage background;

Follow the same steps to load image as we did for Ball but this time we will use getClass() which points to the current object’s class

try {
  background = ImageIO.read(
    getClass().getResource("background.png"));
} catch (IOException | IllegalArgumentException e) {
  background = null;
}

Step 4: Draw a Background Image

Follow similar steps to draw the background image as we did for the Ball sprite but draw inside paintComponent()

 if (background!= null) {
g2.drawImage(background, 0, 0, WIDTH, HEIGHT, null);
} else {
	g2.setColor(FG);
}

Step 5: Add a wall boundary

Now we will stop the ball from moving outside the screen. This is not full wall collision yet. This is just screen-edge collision.

Check the left edge of the screen. If the ball tries to move past it, set the x position to 0. Check the right edge using the constant for WIDTH 

if (x < 0) {
    x = 0;
  }
  if (x + 2*radius > GameComponent.WIDTH) {
    x = GameComponent.WIDTH - 2*radius;
  }

We do not use x == 0 for collision checks. Movement can skip over exact values. Use <=, >=, <, or > instead.

Do you see why we need to add diameter to x and subtract diameter from WIDTH?
