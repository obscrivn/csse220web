Milestone check (In-Class)

  • Your team must be able to run the Ball demo (Tier 2 scaffold).
  • You must demonstrate movement using keys (A/D) and reset behavior.
  • Each team member should be able to explain how Ball, GameComponent, and GamePanel interact.
  • You should be able to explain how input (keys/buttons) connects to object movement.

Team workflow (Required)

  • Work in pairs using pair programming:
    • Driver: types and writes the code
    • Navigator: reviews, guides, and thinks ahead
  • Switch roles regularly during the session.
  • Both partners must understand the code being demonstrated.

Project setup (Required)

  • Install GitHub Desktop and log in.
  • Create a team repository for the project.
  • Add all team members as collaborators.
  • Assign one team member as the submission captain.
  • Push your current scaffold code to the repository.
  • Verify that all team members can access the repository.

Team coordination

  • Schedule regular team meetings outside of class.
  • Create and share a Google Doc for Milestone 0 planning and submission.
  • Use the document to track ideas, responsibilities, and progress.
