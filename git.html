Card 1: Install Git (Windows)

You do not need to use Bash or the command line for this course. Git just needs to be available so Eclipse can use it.

Step 1: Check if Git already works in Eclipse

  1. In Eclipse, open:
    Window ‚Üí Show View ‚Üí Other ‚Üí Git ‚Üí Git Repositories
  2. If the Git Repositories view opens, you are done.
    Do not install anything.

Step 2: Install Git (only if the check failed)

  1. Install Git for Windows:
    https://git-scm.com/download/win

  3. In Eclipse, open:
    Window ‚Üí Show View ‚Üí Other ‚Üí Git ‚Üí Git Repositories
If the Git Repositories view opens, you are done. Do not install additional Git plugins.
Card 2: Create a Git Token (PAT)
  1. Go to GitHub ‚Üí Settings (Fig.1)
  2. Open Developer settings (Fig.2)
  3. Select Personal access tokens ‚Üí Tokens (classic)(Fig.3)
  4. Select Generate new token -> Generate new token (classic) (Fig.4)
    • Name it something like CSSE220-Final (Fig.5)
    • Select scope: repo only
    • Set an expiration (90 days is fine)
    • Click Generate token
  5. Copy the token and save it somewhere safe (Fig.6)
You will not be able to see this token again after leaving the page. Treat it like a password.
Card 3: Cloning Game Template Repo (Captain only)

Only the team captain does this step first. This creates your team‚Äôs GitHub repository using our official starter template.

Starter template: CSSE220-Final-Project-Starter
Do not click the green Code button. You want Use this template.
  1. Open the starter template link above. Click Use this template ‚Üí Create a new repository.
  2. Repository name: use your team name (example: csse220-final-team-07).
    Set visibility based on course policy (public/private as instructed).
  3. Click Create repository. You now have the team repo (your own copy).
  4. Add teammates as collaborators: Settings ‚Üí Collaborators and teams ‚Üí Add people.
  5. Tell your teammates the URL of the new team repo. Everyone else will follow Card 4 to clone into Eclipse.
Common mistake: If teammates clone before you create the team repo, they may end up in the wrong repository. Make sure the team repo exists first.
Card 4: (All Team Members): Clone the Team Repo into Eclipse

Do this after your captain finishes Card 3 and shares the team repo link. You are cloning the team repository (not the starter template).

You will need your PAT: When Eclipse asks for a password, use your Personal Access Token (PAT) (not your GitHub password).

Steps to import

  1. Open your team repo on GitHub. Click the green Code button and copy the HTTPS URL.
  2. In Eclipse: File ‚Üí Import‚Ä¶ ‚Üí Git ‚Üí Projects from Git
  3. Choose Clone URI and click Next.
  4. Paste the URL into the URI field. (Host + repository path should autofill). Click next
  5. Authentication:
    • User: your GitHub username
    • Password: paste your PAT
    • (Optional) check Store in Secure Store. It will remember your PAT
    Click Next.
  6. Branch selection: should be main. Click Next.
  7. Local destination: keep the default folder (recommended). Click Next.
  8. Choose: Import existing Eclipse projects. Click Next.
  9. Select the project and click Finish. You should now see the team project in your Package Explorer.
  10. Run it once: Right-click src/app/MainApp.java ‚Üí Run As ‚Üí Java Application.
