GitHub Guide

Team flow (read this first)

  1. Everyone: Install Git on your computer
    Follow Card 1
  2. Everyone: Sign in to GitHub and create a Classic Personal Access Token (PAT)
    Follow Card 2
  3. Team Captain only: Create the team repository from the starter and add team members
    Follow Card 3
  4. All other team members: Clone the team repository into Eclipse
    Follow Card 4
  5. Everyone: Learn how to use Git inside Eclipse (branches, commits, push, PRs)
    Follow Card 5
Do not skip ahead. Each step assumes the previous one is completed.

Card 1: Install Git (Windows)

Card 1: Install Git (Windows)

You do not need to use Bash or the command line for this course. Git just needs to be available so Eclipse can use it.

Step 1: Check if Git already works in Eclipse

  1. In Eclipse, open:
    Window → Show View → Other → Git → Git Repositories
  2. If the Git Repositories view opens, you are done.
    Do not install anything.

Step 2: Install Git (only if the check failed)

  1. Install Git for Windows:
    https://git-scm.com/download/win
    On that page, click “Click here to download the latest x64 version of Git for Windows.”
    Accept the default settings during installation.
  2. Restart Eclipse after installing Git.
  3. In Eclipse, open:
    Window → Show View → Other → Git → Git Repositories
If the Git Repositories view opens, you are done. Do not install additional Git plugins.

Card 2: Create a Classic PAT

  1. Go to GitHub → Settings
  2. Open Developer settings
  3. Select Personal access tokens → Tokens (classic)
  4. Select Generate new token -> Generate new token (classic)
    • Name it something like CSSE220-Final
    • Select scope: repo only
    • Set an expiration (90 days is fine)
    • Click Generate token
  5. Copy the token and save it somewhere safe
You will not be able to see this token again after leaving the page. Treat it like a password.
  1. Copy the repo HTTPS URL from GitHub.
  2. In Eclipse: open Git Repositories view.
  3. Click Clone a Git Repository and paste the URL.
  4. When asked for a password: paste your PAT.
  5. Import project: right-click repo → Import Projects…

Option B (if you use SSH): copy SSH + clone

  1. In GitHub settings, add your SSH public key.
  2. Copy the repo SSH URL.
  3. Clone via Eclipse Git tools.
If you don’t already have SSH set up, use HTTPS + PAT. It’s faster for this class.

Card 5 — Branches + using Git inside Eclipse

To avoid merge chaos: one feature per branch, merge via PR.

Branch naming

feature/player-movement
feature/map-loading
bugfix/null-checks

Daily workflow

  1. Switch to main → Pull.
  2. Create/switch to your branch.
  3. Make a small change → Commit.
  4. Push your branch to GitHub.
  5. Open a Pull Request → teammate reviews → merge.
Pro tip: If two people edit the same file all day, you’ll get conflicts. Split responsibilities.

Card 6 — Common issues (quick fixes)

“Authentication failed”

  • You used your GitHub password instead of a PAT.
  • Your PAT expired or doesn’t have repo permissions.
  • Eclipse cached old credentials (restart Eclipse; re-enter PAT when prompted).

Red X / project won’t build after cloning

  • Project → Clean
  • Right-click project → Properties → Java Build Path → ensure src is a source folder

Merge conflicts

  • Don’t spam-click “accept all”. Compare changes line-by-line.
  • After resolving: run the project → commit the resolution.
  • Merge smaller PRs more often to avoid giant conflicts.
If you get stuck: screenshot the error + show your Git view in Eclipse. That’s usually enough for a TA to diagnose fast.
