✅ Null / Empty / Blank Checks

Before you do anything else in a method, check for null and empty inputs. JUnit tests often hit these edge cases first, and the rest of your logic is only correct if the input actually exists.

First rule of methods: handle null and empty inputs first.

Type Null? Empty? Blank? Notes
String s == null s.isEmpty() s.isBlank() "" is empty; " " is blank (but not empty). Always null-check before calling methods.
Array arr == null arr.length == 0 — Null-check before using arr.length.
ArrayList list == null list.isEmpty() — Null-check before calling isEmpty().
HashMap map == null map.isEmpty() — If you’re checking a key: map.containsKey(key). Null-check first.

✨ Safe patterns (what JUnit expects you to do)

// String
if (s == null || s.isEmpty()) {
    // treat as missing / invalid text
}

// Array
if (arr == null || arr.length == 0) {
    // nothing to process
}

// ArrayList
if (list == null || list.isEmpty()) {
    // nothing to process
}

// HashMap
if (map == null || map.isEmpty()) {
    // nothing to process
}

Important rule: check null first, then call methods (isEmpty(), isBlank(), etc.).
