🧠 Java Basics & Big Picture

What exactly is Object-Oriented Programming?
OOP is about modeling programs as objects that bundle: data (fields) + behavior (methods). Think: game characters, bank accounts, Minecraft blocks.

Why do we need programs?
To automate tasks, model systems, and solve problems at scale. Computers don’t “think” → programs tell them exactly what to do.

Why do we need public static void main?
It’s the JVM’s entry point. This is where execution starts. No main → nothing runs.

Is static for pure functions?
Not exactly. static means the method belongs to the class, not an object. Some static methods can be a pure helper function (like Math).

📦 Variables, Types, and Memory

How many bits does each data type store?
int: 32 bits
double: 64 bits
char: 16 bits (Unicode)
boolean: JVM-dependent. Conceptually, it represents a single bit of information (true or false) but in practice, it occupies a byte which is the smallest addressable unit of memory.

How does char work?
A char stores a Unicode number under the hood. Example: 'A' → 65.

When do we use single vs double quotes?
'a' → char (single character)
"a" → String (object)

How do you cast String ↔ int?
Integer.parseInt("42")
String.valueOf(42)

🧱 Arrays vs ArrayLists (Big One)

Is there an easy way to remember the syntax difference between arrays and ArrayList?
Arrays use square brackets like arr[index]. ArrayLists use methods like list.get(index) and list.add(value).

How do we add values to arrays?
Arrays have fixed size, so you cannot use add(). You assign values to positions like arr[0] = 10.

Why would we use arrays if ArrayList exists?
Arrays are simpler, faster, and use less memory. ArrayList is more flexible because it can grow.

Is ArrayList similar to Python lists?
Yes. Both store dynamic collections and allow adding items with methods like add() in Java or append() in Python.

Should we use ArrayList if the length is fixed?
Usually no. If you know the size ahead of time, arrays are simpler and slightly more efficient.

Can an ArrayList store arrays?
Yes. Arrays are objects in Java, so something like ArrayList<int[]> is valid.

Why does ArrayList<int[]> work without using Integer?
Because the ArrayList stores the array object itself. The array int[] is already an object even though int is a primitive.

What are the most useful ArrayList methods?
Common ones include add(), get(), set(), remove(), size(), and contains().

Why does ArrayList store objects instead of primitives?
ArrayList is part of Java’s Collections Framework, which works with objects. Generics in Java (the <T> syntax) only support reference types, not primitives. Because of this, you cannot store int, double, or boolean directly. Instead, Java provides wrapper classes like Integer, Double, and Boolean that represent those primitive values as objects.

What is autoboxing?
Java automatically converts primitives into their wrapper objects when needed. For example, if you write list.add(5) in an ArrayList<Integer>, Java automatically converts 5 into an Integer object behind the scenes. This automatic conversion is called autoboxing.

Does this mean ArrayList is slower than arrays?
Slightly. Arrays store raw primitive values directly in memory, while ArrayLists store objects, which adds a small amount of memory and overhead. In practice the difference is usually small, but arrays are still preferred in performance-critical algorithms.

Why use arrays if ArrayLists are more flexible?
Arrays are:

  • Faster
  • Use less memory
  • Fixed-size (predictable)

Which runs faster?
Arrays. No resizing, no boxing/unboxing.

Can arrays store objects?
Yes. Example: Player[] players

Why can’t ArrayLists store primitives?
Generics only work with objects. Primitives use wrapper classes instead.

Do wrapper classes emulate primitives?
They wrap them. Java auto-boxes and unboxes when needed.

Why can ArrayLists change size but arrays can’t?
Arrays are fixed blocks of memory. ArrayLists secretly create new arrays when they grow (!).

Why is .length a field but .size() a method?
length is a field used for arrays, while .size() is a method used for Collection classes (like ArrayList, Set, Map, etc.)is built into arrays.

🧩 Mixing Arrays & ArrayLists

Can you have an array of arrays?
Yes → 2D arrays.

Can an ArrayList store arrays?
Yes → ArrayList <int[]> list = new ArrayList<>();

Now each element of the ArrayList is an entire array.How do you add an array to an ArrayList? 

ArrayList<int[]> list = new ArrayList<>();

int[] nums = {1, 2, 3};
list.add(nums);
Why doesn’t ArrayList need a wrapper class for int[]? Short answer: Because int[] itself is already an object.

Can you convert between them?
Yes, but usually requires a loop (or helper methods like Arrays.asList).

Should we always use ArrayLists?
Nope. Use:

  • Arrays → fixed size, performance
  • ArrayLists → flexible size, convenience

🗂️ HashMaps & Dictionaries

Is a HashMap the same as a Python dict?
Conceptually yes: key → value lookup. Java HashMaps are typed and compiled; Python dicts are dynamic.

What does “hash” mean?
A hash converts a key into a number (hash code) used to find where the value lives.

Why are HashMaps so fast?
Lookups are ~O(1) on average, no searching through a list.

What are HashMaps most useful for?
Counting, fast lookup, mapping IDs → objects, caching.

Can a HashMap store objects?
Yes. Both keys and values can be objects.

Can arrays be keys?
Technically yes, but usually a bad idea unless you override equals and hashCode. Note that arrays are special because Arrays do not override equals(), and both == and .equals() compare references (memory address), not contents.

Are there built-in min/max operations?
No, HashMaps are unordered. You must loop.

What does keySet() return?
A Set<K> of all keys.

Why can we write new HashMap<>()?
Type inference —> Java figures it out from the left side.

Can keys repeat?
No. Duplicate keys overwrite the old value.

🔒 Encapsulation, Access & Constructors

Why make fields private?
To protect your object data being modified by accident or exposed (e.g. bank account, password).

Why getters/setters instead of public fields?
So you stay in control of how data is read or changed.

Can setters update multiple fields?
Yes, very common.

Why use this.name?
To distinguish fields from parameters.

Why can’t a top-level class be private?
Because Java must be able to see it to load it.

Do constructors return anything?
No, they create objects.

When does the default constructor disappear?
When you write any constructor.

How does constructor overloading work?
Java matches parameters by number and type.

Are private methods useful?
Yes, helpers, validation, internal logic.

Wrapper defaults vs primitive defaults?
Wrappers (objects) → null
Primitives → 0 / false etc

🧠 Objects, References & Design

Is null a pointer to address 0?
No, null in Java means "points to nothing" (no memory location assigned yet).

Difference between reference and pointer?
A reference lets you access an object. A pointer gives direct access to memory (hidden by JVM). In String s = new String("hi");

  1. s is the reference,
  2. the actual String object is in heap memory,
  3. the reference is an internal JVM value and you cannot see, print, or inspect it

How do you copy an object?
To create a new object with the same data (not copying the reference), use a copy constructor or copy method logic. A copy constructor creates a new object from another object. A copy method asks an existing object to duplicate itself. Example: Copy constructor vs copy method 

class Player {
    private String name;
    private int score;

    public Player(String name, int score) {
        this.name = name;
        this.score = score;
    }

    // Direct copy
    public Player(Player other) {
        this.name = other.name;
        this.score = other.score;
    }

    // Constructor chaining
    public Player(Player other) {
        this(other.name, other.score);
    }

    // Copy method
    public Player copy() {
        return new Player(this);
    }

    public static void main(String[] args) {
        Player p1 = new Player("Alex", 10);

        // Copy using copy constructor
        Player p2 = new Player(p1);

        // Copy using copy method
        Player p3 = p1.copy();

        // All are different objects
        System.out.println(p1 == p2); // false
        System.out.println(p1 == p3); // false
    }
}

Why are Strings objects?
Strings are immutable and come with methods like length(), substring(), and equals(). Primitives can only store values.

🎭 Overriding, static & final

Why does toString() work without writing it?
It’s inherited from Object.

What is overriding used for?
You use the same method call → but you change the behavior.

Can you override an override?
Yes, unless it’s has the final modifier.

Can static methods be overridden?
No. Because static methods belong to the class, not the object. So the object cannot override them.

What does final do?
Variable → can't be changed
Method → can’t be overridden
Class → can’t be extended

Why are some classes not overridable?
Security, immutability, correctness.

Why doesn’t Java execute chronologically?
Only code that is called from main actually runs. So Java runs by calling methods, not by reading the file top to bottom.

What are common uses of overriding besides toString()?
Overriding is used when different objects should respond differently to the same method call.

  • equals() —> define when two objects are considered equal
  • hashCode() —> required when using HashMap or HashSet
  • draw() / update() —> different behavior in games or GUIs
  • move() / attack() —> game characters behave differently
  • compareTo() —> custom sorting logic
