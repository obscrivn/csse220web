CSSE 220 – Student Q&A

Answers to common student questions (updated as we go!)

← Go Back to Schedule
1. What advantages does Java have over other modern programming languages?
“Modern” isn’t quite accurate – most popular languages like Java, Python, and C++ are decades old. Java stands out because it’s portable (runs anywhere with a JVM), strongly typed, secure, and has true multithreading support (unlike Python’s limited threading).
2. Why did String convert like the char to numeric?
When you do math with char, Java uses the character’s numeric Unicode value.
3. How does interface work / extends?
An interface defines methods a class must implement - the keyword is implements; the keyword extends is used for inheriting from a parent class. (We’ll cover in detail in the OOP unit.)
4. What is the difference between Javadoc and comment?
Javadoc is a special comment (/** ... */) that generates documentation; regular comments (// or /* ... */) are just for humans reading code.
5. Will most classes be organized like this?
Yes – most Java classes follow a pattern: fields → constructors → methods.
6. Can new types be created?
Yes, you can define your own classes, which become new types.
7. What is the primary use for arrays?
Arrays store multiple values of the same type in a single variable.
8. I am confused how the parts are labeled.
We’ll review class structure terms (field, constructor, method) in upcoming classes — check the schedule.
9. I want to know more about heap space.
Heap is memory where objects are stored; managed by the JVM. (We’ll discuss more in the memory management section.)
10. Memory aspect of instance and local variables?
Instance variables live on the heap with the object; local variables live on the stack during method execution.
11. Long and short types are common? Do they prefer doubles or are they preferable for some uses over others?
int and double are most common; long and short are used in special cases where memory or very large numbers matter.
12. Curious when you add two chars why print behaves like that?
Adding chars gives a numeric sum of their Unicode values, not string concatenation.
13. Eclipse versus IntelliJ? Is Eclipse popular in industry also?
Both are widely used – IntelliJ is very popular in industry, but Eclipse is also used in many companies and is especially common in education. We’ll use Eclipse to keep everyone on the same page and avoid confusion, while still building transferable skills.
14. Why does OneDrive cause many problems on Eclipse?
OneDrive sync conflicts with Eclipse’s workspace files – it’s better to keep projects outside of synced folders.
15. Why Java is used for websites apps and not HTML?
HTML is for structure/layout of webpages; Java is a programming language that adds logic, processing, and server-side features.
16. How public and private classes affect code?
Public classes are accessible from anywhere; private classes/members restrict access to within the class.
17. Field, constructors, methods.
Field: stores data · Constructor: creates and initializes objects · Method: defines behavior.
18. What is the difference between primitive and reference data types?
Primitives store simple values directly (int, char, etc.); references store addresses of objects.
19. ASCII chars, constructors?
ASCII values are numeric codes for characters; constructors are special methods to build objects.
20. Why adding chars results in addition of numbers from ASCII table? Is it an established rule?
Yes – Java automatically promotes char to numeric int during arithmetic.
21. How do we memorize char numbers for adding?
You don’t need to memorize – just remember characters map to Unicode numbers; use casting (int) 'A' to check.
22. How to debug the code if error is in JDK/JRE/JVM?
Usually update or reinstall Java; most beginner errors are in code, not the JDK/JRE itself.
23. Polymorphism — what to know more about polymorphism?
Polymorphism lets one interface or superclass be used with many implementations (e.g., calling the same method on different object types). (We’ll cover in detail in the inheritance/polymorphism unit.)
24. Why do we need to uncheck module-info.java and package-info.java when creating projects in Eclipse?
module-info.java is useful for large enterprise apps where you need to define which packages are exposed to other modules. Our projects are single-module, single-package, so module-info.java isn’t needed. package-info.java is just documentation: It’s meant for writing Javadoc for an entire package. In CSSE 220, we’ll focus on class-level Javadoc — package docs are overkill.
