CSSE220: Object-Oriented Software Development Syllabus

Catalog Description

Object-oriented programming concepts, including the use of inheritance, interfaces, polymorphism, abstract data types, and encapsulation to enable software reuse and assist in software maintenance. Recursion, GUIs, and event handling. Use of common object-based data structures, including stacks, queues, lists, trees, sets, maps, and hash tables. Space/time efficiency analysis. Testing. Introduction to UML.

Course Prerequisites

CSSE 120 - Introduction to Software Development (or demonstrated programming efficiency)

Credit Hours

4

Official Learning Outcomes

Students who successfully complete this course should be able to:

  • Develop software that incorporates techniques such as inheritance, class hierarchies, interfaces, polymorphism, casting, exceptions, collections, event-driven GUIs, use of large-scale API packages (e.g., GUI libraries), and recursion
  • Perform steps of the software development cycle effectively including design (as expressed by UML class diagrams), documentation before coding, unit and system testing
  • Explain the implementation of linked lists and analyze the asymptotic worst-case run time of simple programs
  • Predict the performance of simple algorithms (search and sort) given their asymptotic worst, best, and average case run times
  • Select basic data structures (arrays, linked lists) based on typical operation time complexity
  • Work in teams of 2–4 students on a software development project involving multiple iterative development cycles, demonstrating effective team roles, decision making, division of labor, and conflict resolution

Specific Learning Objectives in the Course

Key: HOTS - Higher Order Thinking Skills; LOTS - Lower Order Thinking Skills.

After successfully completing CSSE220, students will be able to:

  • Create an instance of a common Java API class (HOTS) and write code to invoke its methods
  • Write operations utilizing variables of types such as String, ArrayList, HashMap, and built-in arrays (both 1D and 2D)
  • Write operations using loops (for, while, do-while) and conditional statements (if-then, if-then-else, conditional operator ?:)
  • Identify and explain the Five CSSE220 OO Design Principles (LOTS) and redesign UML designs accordingly (HOTS)
  • Create UML designs and write programs from scratch that follow these designs (HOTS)
  • Create classes using inheritance, interfaces, abstract classes, and demonstrate proper class design techniques (HOTS)
  • Define key terms such as polymorphism, inheritance, coupling, cohesion, encapsulation, etc. (LOTS)
  • Create unit tests based on English descriptions of behavior (HOTS) and write recursive operations to solve problems (HOTS)
  • Utilize graphics libraries (e.g., Java Swing) to design and program GUIs that respond to various events (HOTS)
  • Identify and refactor code duplication and analyze the Big-O performance of algorithms and data structures (HOTS)
  • Design and implement a large program from scratch, working as part of a small team (HOTS)

Grading Overview

Over the quarter, the following will be administered:

  • Daily In-Class Coding Labs
  • Homework Assignments
  • Daily In-class Worksheets/Quizzes
  • Two Exams
  • Final project

Each assignment or exam has an associated point value and rubric. The overall letter grade is determined by the weighted average of the following categories: 

Category                                Weight (%)
------------------------------------------------------
Programming Homework                    15%
Non-Programming Homework                15%
In-class Worksheets/Quizzes             10%
In-class Labs                           15%
Final Project                           20%
2 Exams                                 20%
Attendance                               5%

Final Project Average

The Final Project is critical. You must have at least a 'D' average for the overall project grade to pass the course. An 'F' average on the project (< 60%) results in an F for the course.

Exam Grading Averages

Each exam is out of 100 points. Important rules include:

  • You must earn at least one exam score ≥ 70% to be eligible for a course grade of C or higher
  • You must have at least a 'D' average over all exams to pass
  • An 'F' average over the exams (< 60%) results in an F for the course
  • You must demonstrate that you can individually write and debug Java programs at the course level
